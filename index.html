About Me

I am a programmer currently studying Computer Games Programming BSc(Hons) at Staffordshire University. Gaming has been an interest of mine from an early age. I played on the Super Nintendo and Playstation 1 with my father and fell in love with gaming whilst gaining a curiosity as to how they worked under the hood.

I see myself as a very sociable person and often take the mantle of leader role in group projects. This has aided myself in roles where social and leadership skills become necessary for great team based projects along with developing organisational skills.

Programming is a passion but also a challenge. I tend to conceptualise programming as a jigsaw puzzle, where programming creates the jigsaw pieces used within the overall vision. Working hard to solve each puzzle makes each challenge more rewarding, making me love the challenge that all aspects of programming provide.

I enjoy being creative and thinking of new innovative ideas for games; being a games programmer helps me express that into an art form I love. I'm always looking to improve my skills and knowledge of programming in all departments. I always want to better myself and the team around me to be the best we can be, so I am very open to feedback on my work and the techniques I use.

Recent Work

Fierce Engine - Vulkan

(Oct 2022 - Present)

Vulkan is a Graphic API that is Khronos' counter to DirectX it has various benefits to OpenGL, one being the support of Ray Tracing and lower overall CPU overhead. The downside of this is that Vulkan's design is explicit; meaning that the graphics driver has to do less guesswork ultimately leaving more work for the programmer


Follow my journey from learning Vulkan, to ray tracing with Vulkan for global illumination

Fierce Engine Part 1 - DirectX 11

(September 2022 - Present)

Returning to University after my placement year, I returned to working with DirectX 11. Instead of using my previous work, a new framework was used. This exploration of DirectX had a focus on advanced graphical techniques, such as parallax occlusion mapping, shadows, deferred rendering and screen space effects.


For this project, I became accustom to ImGui to allow for debug control within the scene, lights needed to be controlled as many graphic tecnhiques were based on the lights position/direction. Other aspects was to swap what type of rendering was used and highlighting techniques that would otherwise be blended together.


This exploration helped me develop a thorough understanding of graphics pipelines and the shader stages, that has vastly helped in my later projects like the Hudson Engine and my Vulkan Engine.

Technical Games Production

(January 2020 - June 2020)

Technical Games Production was a six-month project where a team and I would work with Unreal Engine 4 to build a game. There were no guidelines so, we had full creative freedom. I had never worked with Unreal Engine, other than with blueprints, so I took the time to learn Unreal with C++ and how to provide an interface to the editor. For the game, the team decided to blend ideas of Tomb Raider and Indiana Jones with a rogue-like setting.


My focus was creating the PCG element of the game along with how rooms were generated. These rooms were a base layer that other members could create variants of these rooms and the PCG scene generator would place rooms within the generated structure based on weightings, the higher the weight more common the room. I worked closely with Scott, who developed the puzzle elements to allow each room to be self-contained and have no effect on external puzzle elements through the use of delegates.

True Engine Part 2 - DirectX 11

(January 2019 - June 2019)

For the second semester, we expanded upon the existing framework to create a physics system to create a true to life simulation of forces. These included friction, torque, gravity, buoyancy and collision handling. Unlike the first semester, part two of the engine was more focussed on an applied engine rather than graphics techniques. This allowed me to develop an understanding of how systems link together and are expected to work together.


There were areas I would want to improve on this project. For example, I would have axis aligned collisions combine with torque as orignally it had no affect on the collision box. This would allow it to then correlate with being able to collisions have physics working together to determine what is the correct simulation based upon the type of force applied.

True Engine Part 1 - DirectX 11

(September 2018 - December 2018)

My first introduction to Graphics APIs was DirectX 11. In my first year I learnt how to draw vertices, render textures and create a physics system that would work with the 3D environment. Developing this allowed me to gain an interest in graphics and engine development.


For the first semester, I learned how shaders work and the integral parts of a graphics pipeline along with how model, view, projection matrices worked. We were given a framework to build from, where we could change the camera based on input, have a billboarding technique working, textures, moving objects based on a tick and a skybox.


Although primitive looking back on my work, it laid a foundation for me to become intersting in graphics development and how the pipeline can be exposed for an engine allowing games to be made. The theory learnt through the my first interaction was brought forward into the second semester.

Soul Heart

(September 2018 - December 2018)

SoulHeart was a project for my first year of university, which I used SDL 2 to make a game using C++. The basic premise was to learn how to make features seen in Pacman and then develop it further. The main game within the program is SoulHeart. SoulHeart is a procedurally generated game using tilemaps along with placing AI that you can fight and end goals within the level. A high-score system was also introduced using a text file to store the data.
