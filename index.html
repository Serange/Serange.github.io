About Me

Shadow Of The Rake

Shadow of the Rake was my first "REAL" game ever made. It was a horror game developed in Unreal Engine as a college project however, there was a games tournament on so myself and my team were planning on taking the original project and further extending it to be the best game developed. I was the lead programmer, Seth was to create the map in Unreal Engine, Mark was to write the story and Nathan was to make the menu elements. With doing this project I had one too many sleepless nights and gallons of coffee working on a laptop to perfect this game to my liking. I learned how to creat UI elements, create Ai behaviour trees have Ai attack you. Have animations blend between each other and to create animations for Unreal Engine. The hardest part about the project was learning how to craft Ai, with changing its animations according to what it was doing within the behaviour tree alongside playing sounds and randomly roaming a navmesh map. I hope you enjoy the video below and see what fueled my passion for games programming.

SoulHeart

SoulHeart was a project for my first year of university, for this project we used S2D to make a game using C++, the basic premise was to learn how to make features seen in pacman then develop it further. The main game within the program is SoulHeart, SoulHeart is a randomly generated map using tilemaps the proceduraly generated map code was developed by myself and does not use an third party plugin/code. The hardest part developing the game was making sure the map is playable whilst generating the map. The program first randomly generates the tile map, then performs a series of checks to see if the end is reachable and the key is reachable. This took countless hours to perfect but persisted with the feature so it could be unique in each playthrough. Other aspects of the game was killing AI, finding a key and reaching the gold block and the storing of highscores to a .txt file. Take a look hope you enjoy.

True Engine

The True Engine was a project in my 2nd year of university, using Direct X 11, I was tasked with building a game engine that used lighting was able to load obj models, alongside multiple optional graphical features. The first vieo below is the engine without a physics engine alongside the base engine. In this version of the True Engine billboarding was implemented, a technique used to reduce strain on GPUs. Another graphical feature was having multiple shaders, the main shader for the engine used phong shading, per pixel lighting, it calculated the diffuse ambient and specular, it calculated and applied it on a pixel level. The second shader was without lighting calculations. Alongside these graphical elements were basic collision for the mines and the ability to control models within the scene if programmed into the application
